In Review R-01

Hybrid Neural Accelerator

Active Research Feb 2026 – Present GLSVLSI 2026 (Late Breaking Session)

A reconfigurable LUT / logarithmic compute engine for edge-AI inference — bridging Python-trained neural networks to a custom IP block on AMD Artix-7 silicon, with PPA optimisation as a first-class design constraint.

TypeFPGA hardware accelerator
TargetArtix-7 XC7A200T (215 K LUTs, 740 DSPs, 13 Mb BRAM)
ToolingVivado HLS · Vitis · AXI4
SubmissionGLSVLSI 2026, Late Breaking Session — Approved

Overview

Edge inference is bottlenecked by the cost of multipliers. As precision scales down — INT8, INT4, ternary, binary — the relative overhead of memory traffic and control logic dominates the silicon budget. This project asks whether a hybrid compute primitive — switching between LUT-based and logarithmic-domain arithmetic on a per-layer basis — can reclaim area, power, and timing simultaneously.

The accelerator is co-designed with a sensitivity-driven quantisation pipeline (the subject of the GLSVLSI 2026 submission) so that layers most tolerant of approximation are mapped to the cheapest compute path automatically.

Architecture

  • Compute Engine (SYS-01) — Reconfigurable LUT / log datapath written in Vivado HLS. Switches mode based on layer descriptor; INT8 → INT4 paths share storage.
  • RTL Backend (SYS-02) — HLS-synthesised RTL placed on Artix-7. Focus on timing closure, BRAM/LUT/DSP balance, and clock-domain hygiene.
  • AXI4 Interconnect (SYS-03) — Memory-mapped AXI4 between the custom IP and an embedded ARM core; DMA-driven weight streaming and burst-aligned activations.
  • Quantisation Pipeline (SYS-04) — Python-side sensitivity analysis selects per-layer precision; descriptors are baked into the bitstream metadata at synthesis.
  • Verification (SYS-05) — Bit-accurate C++ model paired with cocotb testbenches; coverage-driven regression on a representative DNN slice.
  • PPA Telemetry (SYS-06) — Vivado reports parsed into a sweep dashboard for reproducible design-space exploration over precision × topology.

Current Status

  • HLS compute primitive — First-pass LUT / log datapath synthesised; INT8 inference correct against reference model.
  • RTL prototype — Block placed on Artix-7 with timing closure at the target frequency for the first compute tile.
  • AXI4 integration — Memory-mapped path from ARM core to custom IP validated on small workloads.
  • Publication — Companion paper "Quantization-Aware Training for Efficient Edge DNN Deployment" approved for GLSVLSI 2026.
